# 독쌤 입학상담 엔진 — 전체 설계도 (ENGINE_DESIGN.md)

> 목적: 흩어진 자료(3개 MD + 커스텀 설문 + 로드맵 도구)를 **하나의 일관된 웩슬러 기반 상담 엔진**으로 통합하기 위한 설계 기준 문서.
> 성격: 코드/내용을 채우기 전에 **무엇이 어디서 무슨 역할을 하고 어떻게 연결되는가**를 먼저 확정하는 골격.
> 원칙: 큰 시스템은 설계 먼저. 이 문서가 확정돼야 이후 MD 정리·코드 구현이 흔들리지 않는다.
> 상태: **v1.0 (2026-06 설계 확정)** — 스탠다드/커스텀 2-tier 구조 + Apps Script 저장 + 여덟 겹 철학 반영 완료. 다음은 구현 단계.

---

## 0. 한 줄 요약

> 학부모가 **커스텀 설문**에 아이 정보(진도·환경·목표·웩슬러 6지수)를 넣으면 →
> **해석 엔진**이 웩슬러를 수학 능력으로 번역하고 L0~L4 위치를 판정하고 균형을 진단한 뒤 →
> **지식 근거**(소마·필즈·프리미어 기준 + 제도 변화)에 비추어 →
> **단정 없이** 가능성·주의점·확인질문·1~3개월 우선순위를 담은 상담 멘트를 만든다.

## 0-A. 스탠다드 / 커스텀 2-tier 구조 (2026-06 확정)

> 엔진은 **두 층(tier)**으로 나뉜다. 무료 공개(스탠다드)와 유료 상담(커스텀). 챗봇은 "맛보기", 정교한 분석은 커스텀=카톡 상담으로 유도한다.

| 구분 | **스탠다드 (공개·무료)** | **커스텀 (유료·상담)** |
|---|---|---|
| 입력 | 진도만 (나이·사고력·교과·연산) | 스탠다드 진도 + **웩슬러 6지수·환경·과제관리** |
| 웩슬러 | **점수 입력 없음.** "유동추론이 어쩌구라면 우선 이 부분을 채워주세요" 정도 방향 제시만 | FSIQ·VCI·VSI·FRI·WMI·PSI 입력 → W-001~008 정밀 해석 |
| 출력 | 레벨 판정(L0~L4) + 균형 진단 + **여덟 겹 철학까지 설명** | + 맞춤 로드맵(환경·과제 반영 시점 조정) + 목표 대비 정밀 갭 분석 |
| 형태 | 텍스트 | 유료 API(Claude/Gemini) + **음성 지원** |
| 끝맺음 | "더 정교한 분석은 커스텀에서. 커스텀 신청하시겠습니까?" → **원장 카톡 연결** | 원장 직접 상담 (리드 확보) |

**핵심 원칙:** 스탠다드는 진도만으로 철학까지 주되, **다 보여주지 않는다.** 정밀 분석(웩슬러 조합·맞춤 시점)은 커스텀으로 가려서 상담 신청을 유도한다.

**데이터 연결:** 커스텀은 스탠다드에서 입력한 진도값을 **끌어와야** 한다(재입력 X) → 저장 필요 → 섹션 0-B.

## 0-B. 저장 아키텍처 (2026-06 확정 · Apps Script + Sheets)

> 학부모가 입력한 진도를 **어느 기기에서든 승인번호로 다시 볼 수 있어야** 한다(경우 2). 커스텀도 저장된 진도를 끌어온다. → 저장은 **Google Apps Script + Google Sheets**.

**왜 Apps Script인가 (VM·Supabase 대신):**
- 서버 구축·관리 불필요 (구글이 호스팅). VM 꺼져도 동작.
- 토큰·키 노출 문제 없음 (쓰기 로직이 스크립트 안에 숨음. 프론트엔드엔 URL만 노출 — 안전).
- **어드민 = 구글 시트 그냥 열기.** 권한·횟수를 시트에서 직접 수정. 별도 어드민 페이지 거의 불필요.
- 무료.

**구조:**
```text
[학부모 HTML — GitHub Pages]
   진도 입력 → fetch POST → Apps Script 웹앱 URL
                              ↓
                        Google Sheets에 저장 (승인번호 키)
   불러오기 → fetch GET ?code=GF1234 → 시트에서 찾아 JSON 반환 (자동)
```

**Apps Script 엔드포인트 (2개):**
- `doPost(e)` — 승인번호 + 진도 JSON 받아서 시트에 저장(있으면 갱신, 없으면 추가)
- `doGet(e)` — `?code=GF1234` → 그 행의 진도 JSON 반환

**Google Sheets 컬럼 구조:**

| 승인번호 | 이름 | 연락처 | 진도JSON(스탠다드) | 커스텀JSON | 재작성횟수 | 커스텀권한 | 생성일 | 수정일 |
|---|---|---|---|---|---|---|---|---|
| GF1234 | 홍길동 | 010-… | {age,thinking,school,calc} | {wppsi6,env,task} | 1 | FALSE | … | … |

**규칙:**
- **식별자 = 승인번호** (이미 로드맵 도구 approvalGate에서 받음).
- **2회 재작성 제한** = `재작성횟수` 필드. 재진입 시 "새로 작성 / 기존 불러오기" 물어봄. 새로 작성 시 +1, 2 초과면 차단.
- **커스텀 권한** = `커스텀권한` 필드. 어드민(원장)이 시트에서 TRUE로. 커스텀 페이지 진입 게이트.
- 진도값만 저장하면 **불러오기는 자동** (승인번호로 그 행 읽으면 끝).

**보안:** Apps Script 웹앱 URL은 공개돼도 안전(쓰기 검증은 스크립트 내부). 단 승인번호 없는 요청은 거부. 개인정보는 URL 파라미터 최소화(GET은 code만, 민감정보는 POST body).

## 2. 자료별 역할 분담 (단일 출처 원칙)

> 같은 내용이 두 곳에 있으면 모순·중복이 생긴다. 각 주제는 **단 하나의 출처**만 갖는다.

| 주제 | 단일 출처 (Single Source of Truth) | 다른 곳에서는 |
|---|---|---|
| 웩슬러 지수 해석·프로파일 규칙 | **wppsi_reference.md** | 참조만 (재서술 금지) |
| L0~L4 레벨·진도표·교재 단계 | **로드맵 도구(코드) + KB의 진도 표** | 설문은 목표레벨 선택지로만 사용 |
| 균형 진단 로직(연산/교과/사고력) | **로드맵 도구 코멘트 로직** | wppsi는 "판단 흐름"에서 참조 |
| 소마·필즈·프리미어 응시 기준·철학 | **docssam_counseling_kb.md** | wppsi는 "학원 연결 규칙"에서 참조 |
| 제도 변화·표현 전환 | **market_trend_2026.md** | KB·wppsi는 표현 규칙만 인용 |
| 입력 항목(설문 문항) | **custom_survey.html** | — |

**정리 작업 시 핵심:** 웩슬러 내용이 지금 custom_survey(코멘트)·wppsi 양쪽에 흩어져 있음 → **wppsi를 유일 출처로 통일**, 설문은 "입력만 받고 해석은 wppsi 규칙을 따른다"로 정리.

## 3. 데이터 흐름 — AI 실행 순서 (10단계)

> wppsi_reference 섹션 11의 흐름을 전체 엔진 기준으로 확장.

```text
1. 개월수 확인 (생년월일·검사일 → 실제 개월수, 한국나이)
2. 웩슬러 6지수 프로파일 확인 (FSIQ·VCI·VSI·FRI·WMI·PSI)
3. 지수 간 차이 확인 (W-001~008 프로파일 규칙 매칭)
4. 현재 교재 위치 확인 (킨더~1031, 디딤돌, 문해길)
5. 레벨 판정 (사고력1 진도 → L0~L4 중 best = 가장 가까운 레벨)
6. 균형 진단 (연산→교과→사고력 순서: 역전/불균형/좋은신호)
7. 질문 분류 (소마/필즈/프리미어/황소 중 무엇을 묻는가)
8. 학원 기준 비교 (KB 응시 신호 + 제도 변화 표현)
9. 단정 대신: 가능성 + 강점재료 + 주의점 + 확인질문
10. 1~3개월 지도 우선순위 + 학부모 상담 문장 생성
```

## 4. 핵심 연결 규칙 (자료 ↔ 자료)

### 4-1. 웩슬러 지수 → 수학 능력 (wppsi 섹션 6)
- VCI(언어이해) → 지문 해석·조건 읽기 ("사고력"의 한 축)
- VSI(시공간) → 도형·쌓기나무·공간
- FRI(유동추론) → 규칙·패턴 추론 ("사고력/창의성"의 핵심 축)
- WMI(작업기억) → 암산·조건 유지·다단계
- PSI(처리속도) → 시간·쓰기·시험 수행
- FSIQ → 참고만, 단독 판단 금지

### 4-2. 웩슬러 프로파일 → 상담 (wppsi W-001~008)
- 조합으로 해석: FRI높음+VCI높음 / VCI높음+WMI낮음 / FRI높음+연산부족 등
- 모든 규칙의 공통 꼬리표: **"검사는 지도 방식 조정용, 합격 예측 아님"**

### 4-3. 진도 → 레벨 (로드맵 도구 LEVELS)
- 사고력1(킨더팩토→키즈→초등팩토→필즈베이직→1031) 진도로 best 레벨 판정
- shift: L0=0, L1=3, L2=7, L3=10, L4=12 (개월, 클수록 늦게 시작=안정)
- **L0~L2 = 속도 트랙**(소마·필즈 선발 속도) / **L3~L4 = 토대 트랙**(속도 무관, 나중에 잘할 토대)
  - L3 = 필즈 주2회 E2 수준
  - L4 = 안정적 상위권·소마 A반

### 4-4. 연산 기준 (오늘 확정)
- 킨더팩토 구간 = 소마셈 K / 키즈팩토 구간 = 소마셈 P / 이후 기적의 계산법 2~6권

### 4-5. 균형 진단 (코멘트 로직, 오늘 확정)
- 정상 순서: 연산 ≥ 교과 ≥ 사고력
- 연산<교과(역전) → 재설정 경고 (최우선)
- 교과·연산 잡힘 + 사고력만 뒤처짐 → "좋은 신호, 사고력에 시간 할애"

### 4-6. 제도 변화 표현 (market_trend M-001~004)
- "필즈 준비/입테 대비" → "필즈 정도 학원학습을 감당할 실력"
- 시험 폐지 ≠ 선발/실력 기준 소멸
